# TaskMonitor

A Windows desktop application that monitors Excel/CSV spreadsheets for upcoming and overdue tasks, displaying notifications and a live task list in a modern WPF UI.

Built with PowerShell, MahApps.Metro, and MaterialDesign. No installation required — NuGet packages are auto-downloaded on first run.

![TaskMonitor](assets/TaskMonitor/taskmonitor_512.png)

---

## Features

- **Spreadsheet monitoring** — tracks due dates across multiple worksheets in a single Excel/CSV file
- **Smart categorisation** — overdue tasks shown in red/yellow, upcoming tasks in blue; non-working day tasks highlighted in amber, with configurable look-ahead windows per sheet type
- **Working days awareness** — tasks due on non-working days are flagged in amber so you can act early
- **Windows toast notifications** — alerts when tasks are due
- **Taskbar badge** — red overlay count on the taskbar icon
- **System tray support** — optionally minimise/close to tray and restore via double-click
- **Run at startup** — optional Windows startup registration via the Settings window
- **Auto-refresh** — re-checks the spreadsheet every 60 seconds
- **Example spreadsheet** — generates a ready-to-use multi-sheet workbook if you don't have one yet
- **Fully configurable** — choose which columns contain due dates and task descriptions, per sheet

## Requirements

- Windows 10 or 11
- PowerShell 5.1 (built in) or PowerShell 7+
- Internet access on first run (downloads MahApps.Metro and MaterialDesign via NuGet)
- Microsoft Excel (optional) — required for `.xlsx`/`.xls` files (loaded via COM interop); `.csv` files are read with PowerShell's built-in `Import-Csv` and do not need Excel

## Getting Started

### 1. Run the application

Double-click **TaskMonitor.vbs** (launches without a console window), or run directly:

```powershell
powershell -ExecutionPolicy Bypass -File TaskMonitor.ps1
```

On first run, NuGet packages are downloaded automatically to `%USERPROFILE%\PSLibs`.

### 2. Choose a spreadsheet

You will be prompted to either:
- **Load an existing spreadsheet** — browse to your `.xlsx`, `.xls`, or `.csv` file
- **Create an example spreadsheet** — generates a sample multi-sheet Excel workbook (`.xlsx`) to get started

### 3. Map your columns

For each worksheet, select:
- **Date column** — the column containing task due dates
- **Description column** — the column containing task names/descriptions

Settings are saved automatically and remembered between sessions.

## Spreadsheet Format

TaskMonitor expects each monitored worksheet to have a header row with at least:

| Task Description | Due Date | Date Last Performed |
|------------------|----------|---------------------|
| Check server backups | 15/04/2026 | 01/04/2026 |

- **Date column** — supports most date formats (dd/MM/yyyy, MM/dd/yyyy, yyyy-MM-dd, etc.)
- **Date Last Performed** column — updated automatically when you mark a task complete in the app; the **Due Date** column should use a formula referencing this column so the next due date recalculates automatically (this is set up for you in the example spreadsheet generated by the app)
- **Multiple worksheets** — each sheet can represent a different task frequency (Weekly, Monthly, Annual, etc.)

### Look-ahead windows by sheet type

| Sheet name contains | Look-ahead | Example (today = Wednesday) |
|---------------------|------------|------------------------------|
| Weekly | 1 day | Shows tasks due Wed, Thu |
| Annual / 6-Monthly | 28 days | Shows tasks due within 4 weeks |
| Everything else | 7 days | Shows tasks due within 1 week |

## Settings

Open the **Settings** window (gear icon) to configure:

| Setting | Description |
|---------|-------------|
| Spreadsheet path | Browse to change the monitored file |
| Working days | Which days count as working days for highlighting |
| Run at startup | Register TaskMonitor in the Windows startup registry |
| Minimise/Close to tray | Hide to system tray instead of minimising/closing |
| Column mappings | Per-sheet date and description column selection |

Configuration is stored in `%APPDATA%\TaskMonitor\config.ini`.

## File Structure

```
TaskMonitor/
├── TaskMonitor.ps1                 # Entry point
├── TaskMonitor.vbs                 # Silent launcher (no console window)
├── assets/                         # Icons and images
├── lib/
│   ├── Init.ps1                    # WPF library setup, logging, AppUserModelID
│   ├── Config.ps1                  # INI config read/write

│   ├── Spreadsheet.ps1             # Excel/CSV loading, example workbook generator
│   ├── XamlHelpers.ps1             # XAML utilities and theme resources
│   └── Register-TaskbarHandlers.ps1 # Explorer restart / session unlock handling
└── windows/
    ├── MainWindow.ps1              # Main task list window
    ├── SystemTray.ps1              # System tray (NotifyIcon)
    ├── Settings.ps1                # Settings dialog
    ├── ColumnSelection.ps1         # Column picker dialog
    ├── FileChoice.ps1              # Load/create spreadsheet dialog
    └── WorkingDays.ps1             # Working days setup dialog
```
